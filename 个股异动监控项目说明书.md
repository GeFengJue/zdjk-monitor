# 个股异动监控项目说明书

> 更新时间: 2026-04-14

## 一、项目概述

本项目用于自动采集和展示A股个股异动监控数据，包括：
- 重点监控股票列表
- 历史监管记录
- 问询函件

## 二、技术架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   数据源        │     │   GitHub        │     │   用户浏览器     │
│   同花顺API     │ ──► │   仓库          │ ──► │   HTML页面      │
│                 │     │   + Actions    │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │
        ▼                       ▼
   定时采集脚本           存储JSON数据
   (每小时)              自动更新
```

## 三、数据源

### 3.1 API接口

使用同花顺API接口获取数据：

| 接口 | 用途 | 请求方式 |
|------|------|---------|
| GetYDTP_ZDJK_Today | 重点监控今日 | POST |
| GetYDTP_ZDJK_His | 重点监控历史 | POST |
| GetYDTP_WXHJ_His | 问询函历史 | POST |

API地址: `https://apphq.longhuvip.com/w1/api/index.php`

### 3.2 数据格式

```json
{
  "fetch_time": "2026-04-14T00:00:00.000Z",
  "ZDJK_Today": {
    "List": [
      ["股票代码", "股票名称", "开始日期", "结束日期", "类型"]
    ]
  },
  "ZDJK_His": { ... },
  "WXHJ_His": { ... }
}
```

## 四、部署步骤

### 4.1 创建GitHub仓库

1. 登录GitHub: https://github.com
2. 点击右上角 "+" → "New repository"
3. 仓库名称: `zdjk-monitor`
4. 选择 Public
5. 不要勾选 "Initialize this repository with a README"

### 4.2 本地初始化（已完成）

已在本地创建完整项目结构：

```
zdjk_github/
├── zdjk_fetcher.py          # 数据采集脚本
├── update_readme.py         # 自动更新README
├── requirements.txt         # Python依赖
├── README.md               # 项目说明
├── .gitignore              # Git忽略配置
├── data/
│   └── zdjk_data.json      # 数据文件
└── .github/
    └── workflows/
        └── fetch.yml        # GitHub Actions配置
```

### 4.3 推送到GitHub

```bash
cd zdjk_github
git init
git remote add origin https://github.com/GeFengJue/zdjk-monitor.git
git add -A
git commit -m "Initial commit"
git branch -M main
git push -u origin main
```

### 4.4 启用GitHub Actions

推送代码后，GitHub Actions会自动检测 `.github/workflows/fetch.yml` 并执行。

手动触发: 访问 `https://github.com/GeFengJue/zdjk-monitor/actions` → "Run workflow"

## 五、数据更新机制

### 5.1 自动更新

GitHub Actions 配置为每小时自动执行：

```yaml
schedule:
  - cron: '0 * * * *'  # 每小时整点执行
```

### 5.2 手动更新

1. 访问 https://github.com/GeFengJue/zdjk-monitor/actions
2. 点击左侧 "Fetch ZDJK Data"
3. 点击 "Run workflow" → 选择分支 → Run

## 六、数据存储

### 6.1 存储位置

- **仓库**: https://github.com/GeFengJue/zdjk-monitor
- **数据文件**: `data/zdjk_data.json`
- **JSON访问**: https://raw.githubusercontent.com/GeFengJue/zdjk-monitor/main/data/zdjk_data.json

### 6.2 数据保留

- **GitHub仓库**: 永久保留，无时间限制
- **GitHub Actions 日志**: 约90天后自动清理
- **本地浏览器缓存**: 使用 localStorage，最长7天

### 6.3 数据大小

每次采集的数据约 5-10KB，非常轻量。

## 七、网页访问

### 7.1 页面地址

- **本地文件**: 直接用浏览器打开 `重点监控.html`
- **GitHub Pages**: https://GeFengJue.github.io/zdjk-monitor/重点监控.html

### 7.2 页面功能

| 功能 | 说明 |
|------|------|
| 重点监控 | 当前监控中的股票 |
| 历史监管 | 已过期的监控记录 |
| 问询函件 | 风险揭示函PDF链接 |
| 自动刷新 | 每5分钟自动获取最新数据 |
| 剩余天数 | 自动计算解封倒计时 |

### 7.3 加载加速

使用 `corsproxy.io` CDN代理加速GitHub raw文件访问，解决跨域和加载慢的问题。

## 十、修改和维护

### 10.1 修改采集脚本

1. 编辑 `zdjk_fetcher.py`
2. 测试运行: `python zdjk_fetcher.py`
3. 提交推送: `git add -A && git commit -m "更新" && git push`

### 10.2 修改HTML页面

1. 编辑 `重点监控.html`
2. 浏览器测试
3. 推送更新

### 10.3 修改定时任务

编辑 `.github/workflows/fetch.yml` 中的 cron 表达式：

```yaml
schedule:
  - cron: '0 * * * *'    # 每小时
  # - cron: '0 */6 * * *'  # 每6小时
  # - cron: '0 9,15 * * *' # 每天9点和15点
```

## 十一、常见问题

### Q: 数据多久更新一次？
A: 默认每小时自动更新一次，可在GitHub Actions手动触发即时更新。

### Q: 为什么有时候页面加载很慢？
A: GitHub raw文件访问有时较慢，已添加CDN代理加速。如仍慢可刷新重试。

### Q: 历史数据会丢失吗？
A: GitHub仓库中的数据永久保存，不会丢失。

### Q: 可以私有化部署吗？
A: 可以，只需fork仓库并修改数据源URL即可。
